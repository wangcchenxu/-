# 分层递进式排班算法 v4.0 测试指南

## 一、算法概述

新版本实现了完整的**七层分层递进式排班算法**:

1. **第零层**: 数据准备
2. **第一层**: 月度配额计算(宏观均衡)
3. **第二层**: 连续块模式生成(中观结构优化)
4. **第三层**: 逐日精排与冲突消解(微观分配)
5. **第四层**: 全局优化(启发式调整)
6. **第五层**: 统计与记忆更新
7. **验证层**: 结果验证

## 三、关键检查点

### ✅ 正常情况

如果看到以上完整日志,说明算法执行成功。

### ⚠️ 警告情况

如果出现以下警告,需要进一步分析:

```
[警告] 第X天白班人数: 1/2
[警告] 员工Y 实际天数: 14/15
```

这可能表示:
- 约束过于严格,无法找到可行解
- 块模板生成有问题
- 贪心放置失败过多

### ❌ 错误情况

如果出现红色错误:

```
排班算法执行出错: XXX
错误堆栈: ...
```

请记录错误信息,可能的原因:
1. **变量未定义**: 检查函数参数传递
2. **数组越界**: 检查循环边界条件
3. **类型错误**: 检查数据类型转换

## 五、常见问题排查

### 问题1: 看不到新算法日志

**原因**: 浏览器缓存了旧代码

**解决**: 
1. 按 Ctrl + F5 强制刷新
2. 或在无痕模式下测试
3. 检查控制台是否有版本标记: `========== 排班算法 v4.0 ==========`

### 问题2: 算法执行到某一层就停止

**原因**: 该层代码有错误或逻辑问题

**解决**:
1. 查看控制台红色错误信息
2. 检查错误堆栈,定位到具体行号
3. 根据错误提示修复代码

### 问题3: 生成的班表有很多空缺

**原因**: 
- 块模板生成不合理
- 贪心放置失败
- 约束过于严格

**解决**:
1. 检查第二层的块模板是否合理
2. 查看第三层有多少人被后备策略处理
3. 放宽约束条件(如maxConsecutiveDays)

### 问题4: 白夜班不均衡

**原因**: 
- 第一层配额计算有误
- 第三层分配逻辑有问题

**解决**:
1. 检查第一层输出的配额是否正确
2. 检查第三层是否严格执行了配额
3. 查看第五层的统计结果

## 六、性能测试
